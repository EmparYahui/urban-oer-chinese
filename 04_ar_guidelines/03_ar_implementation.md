# ⚙️ AR Implementation Model

## 🧠 Overview

This document defines how AR is operationalised in learning environments.

The model is **tool-agnostic** and focuses on pedagogical function.

---

## 🧩 Implementation levels

### Level 1 — Basic implementation
- QR codes
- static geolocation links
- simple annotation layers

### Level 2 — Intermediate implementation
- mobile AR apps
- interactive overlays
- mapped artefacts

### Level 3 — Advanced implementation
- immersive AR environments
- dynamic spatial narratives
- multi-user interaction

---

## 🧠 Key constraint

Technology must never determine pedagogy.

Pedagogy defines AR use, not the reverse.
