# 04 Task Architecture — Urban Linguistic Landscape OER

## 🧠 Definition

A task is the core operational unit of the system, connecting urban input, mediation, and multimodal output.

---

## 🧩 Task components

Each task must include:

### 1. Urban input
Real-world linguistic or visual data from urban environments.

### 2. Mediation process
Transformation of meaning across languages, registers, or modes.

### 3. Learner output
A structured multimodal artefact.

### 4. OER potential
Output must be reusable or adaptable for educational purposes.

---

## 🧭 Types of tasks

### Observational tasks
Focus: identifying and documenting urban linguistic data.

### Mediational tasks
Focus: translating or reinterpreting meaning.

### Production tasks
Focus: creating multimodal artefacts.

### AR-linked tasks
Focus: spatial contextualisation of meaning.

---

## 🔁 Task lifecycle

1. Input selection  
2. Interpretation  
3. Transformation  
4. Production  
5. Evaluation  
6. Potential reuse

---

## 📌 Design rule

No task is complete unless it produces a traceable and assessable artefact.
