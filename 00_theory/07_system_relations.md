# 07 System Relations Map
## Urban Linguistic Landscape OER (A1–A2)

This document defines the internal relational architecture of the theoretical system. It does not describe components in isolation, but specifies how conceptual modules interact, transform each other, and generate pedagogical outcomes.

The system is non-linear, recursive, and feedback-driven.

---

## 🧠 1. Core relational principle

All components operate under a single systemic principle:

> Learning design, learning activity, and research evidence form a closed-loop system of mutual transformation.

This implies that no module functions independently; each module is both input and output of others.

---

## 🔷 2. Primary system nodes

The theoretical system consists of six core nodes:

- 00 Core Framework
- 01 Urban Linguistic Landscapes (ULL)
- 02 Task and Project-Based Learning
- 03 Multimodality and Mediation
- 04 AR as Mediational Layer
- 05 OER and Open Pedagogy
- 06 Integrated Model

---

## 🔄 3. Dependency structure (vertical relations)

### 3.1 Foundational layer

- 00 Core Framework → defines epistemological conditions for all modules

---

### 3.2 Contextual layer

- 01 ULL → provides empirical environment for all learning processes
- 02 Task/Project-Based Learning → operationalises interaction with ULL

---

### 3.3 Mediational layer

- 03 Multimodality and Mediation → transforms input into semiotic artefacts
- 04 AR → spatially and digitally extends mediation processes

---

### 3.4 Output layer

- 05 OER and Open Pedagogy → converts learner production into reusable knowledge artefacts

---

### 3.5 Synthesis layer

- 06 Integrated Model → consolidates all transformations into a unified pedagogical system

---

## 🔁 4. Horizontal interactions (cross-module dynamics)

### 4.1 ULL ↔ Task/Project-Based Learning
Urban environments are not passive context; they are activated through task design.

### 4.2 Task Design ↔ Multimodality
Tasks define semiotic constraints; multimodality defines expressive expansion.

### 4.3 Multimodality ↔ AR
AR acts as spatial amplifier of multimodal production.

### 4.4 AR ↔ ULL
AR reconfigures urban space as layered semiotic environment.

### 4.5 OER ↔ All modules
All components feed into OER production as both process and outcome.

---

## 🔄 5. Feedback loops (recursive system dynamics)

### Loop A: Learning loop
ULL → Task → Mediation → Output → Reflection → Task redesign

### Loop B: Representation loop
Experience → Multimodal artefact → AR anchoring → Recontextualisation

### Loop C: Knowledge loop
Learner output → Evaluation → OER classification → Reuse → New learning cycle

---

## 🧠 6. System logic constraints

The system operates under the following constraints:

- No module is self-contained
- All outputs are potential inputs for another module
- Pedagogical design is inseparable from research design
- Learning and data production are structurally identical processes

---

## 🌐 7. Emergent property of the system

When all relations are active, the system produces:

> A recursive open pedagogical ecosystem in which urban space becomes a learning infrastructure and learner production becomes reusable knowledge.

---

## 🧩 8. Final structural statement

This is not a curriculum.

It is a relational system architecture for:

- learning design
- multimodal meaning production
- spatial mediation
- and open knowledge generation
